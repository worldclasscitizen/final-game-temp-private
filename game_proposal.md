# 🔫 죽을수록 강해진다 — 총 들고 일진일퇴

> **"죽을 때마다 총이 강해진다. 근데 죽으면 밀린다."**

---

## 1. 원작 소개 — Nidhogg? ROUNDS?

이 게임은 **Nidhogg**와 **ROUNDS**, 두 게임의 핵심 재미를 합쳐서 만든 게임이에요.
각각이 어떤 게임인지 먼저 소개할게요.

### 🤺 Nidhogg (2014)

2D 횡스크롤 **펜싱 대전 게임**이에요.
두 플레이어가 가로로 긴 맵의 한가운데에서 칼을 들고 마주 서요.
상대를 찔러 죽이면, 그 사이에 상대 진영 쪽으로 달려갈 수 있어요.
죽은 쪽은 바로 리스폰되어서 다시 막아서고, 상대 진영 끝까지 먼저 도달하면 승리.
칼싸움을 하면서 밀고 밀리는 **일진일퇴의 긴장감**이 핵심인 게임이에요.

### 🔫 ROUNDS (2021)

1:1 총싸움 게임이에요.
두 플레이어가 총을 들고 서로를 쏘는 단순한 구조인데,
핵심은 **라운드가 끝날 때마다 진 쪽이 카드를 뽑는다**는 거예요.
카드를 뽑으면 내 총이 업그레이드돼요 — 총알이 커지거나, 빨라지거나, 튕기거나, 유도되거나.
라운드를 거듭할수록 총이 점점 괴물이 되어가는 **카드 빌드업**이 핵심인 게임이에요.

---

## 2. 그래서 이 게임은 뭔데?

**Nidhogg에서 가져온 것:** 일진일퇴 구조
- 가로로 긴 맵, 가운데서 시작, 상대 진영까지 돌파하면 승리
- 죽어도 리스폰, 제한시간 없음, 밀고 밀리는 공방

**ROUNDS에서 가져온 것:** 카드 빌드업 + 총싸움
- 무기가 칼이 아니라 총. 마우스로 조준해서 쏨
- 죽을 때마다 카드를 뽑아서 총이 점점 강해지는 시스템

**합치면 이렇게 돼요:**

두 명이 맵 한가운데에서 총을 들고 시작해요.
상대를 쏴서 죽이면 상대 진영 쪽으로 달려갈 수 있고, 끝까지 도달하면 승리.
죽은 쪽은 리스폰 전에 카드를 한 장 뽑아서, 더 강한 총을 들고 다시 태어나요.

이기고 있는 쪽은 빨리 달려가야 하고,
지고 있는 쪽은 점점 무시무시한 총으로 반격할 수 있어요.

**Nidhogg의 일진일퇴 + ROUNDS의 카드 빌드업.** 그게 이 게임이에요.

```
[P1 진영] ◀━━━━━━━━━━━ 맵 ━━━━━━━━━━━▶ [P2 진영]

                    P1 🔫⚡🔫 P2
                     (가운데서 시작!)

P1이 P2를 죽임 →   P1이 오른쪽으로 달려감 →→→
                    P2는 P1 앞에서 리스폰 (카드 1장 획득!)
                    P2의 총이 더 강해진 채로 재대결!

                    ...반복...

P1이 P2 진영 끝에 도달 → 🏆 P1 승리!
```

## 4. 핵심 규칙 정리

### 🎯 기본 규칙

- **플레이어 수:** 2명 (온라인 멀티플레이어)
- **승리 조건:** 상대 진영 끝까지 도달
- **제한 시간:** 없음. 끝날 때까지 끝이 아님
- **원샷 원킬:** 총알 한 발이면 죽음. 체력 같은 건 없음
- **죽어도 끝이 아님:** 리스폰하면서 카드를 뽑고, 다시 싸움

### 🏃 움직임

- 좌우 이동, 점프 가능
- 마우스로 조준, 클릭으로 발사
- 상대를 죽이지 않고 요리조리 피해서 달려가도 OK

### 📷 카메라

- 두 플레이어가 **같은 화면**을 공유
- 카메라는 기본적으로 **이기고 있는 쪽(= 상대 진영에 더 가까운 쪽)**을 따라감
- 카메라 밖으로 벗어난 플레이어는 **자동 사망** 처리 → 달려가는 사람의 앞쪽에 리스폰
- 카메라 이탈로 죽은 경우에는 **카드를 뽑지 않음** (페널티 개념)

### 🃏 카드 시스템

죽었을 때 리스폰 전에 카드를 **1장** 뽑아요. 카드는 랜덤이고, 뽑은 카드의 효과가 내 총에 영구 적용돼요.

**카드 예시:**

| 카드 이름 | 효과 |
|---|---|
| 유도탄 | 총알이 상대 쪽으로 살짝 휘어감 |
| 산탄 | 벽이나 바닥에 튕기면 여러 발로 갈라짐 |
| 고속탄 | 탄속이 빨라짐 |
| 대구경 | 총알 크기가 커짐 |
| 관통탄 | 총알이 벽을 한 번 관통함 |
| 반사탄 | 벽이나 바닥에 1번 튕김 (튕긴 후 다음 충돌 시 소멸) |

카드는 **중복 적용** 가능해요. 유도탄을 두 번 뽑으면 더 많이 휘고, 산탄 + 대구경이면 큰 총알이 여러 개로 갈라지는 거예요. 죽을수록 총이 괴물이 됩니다.

## 5. 캐릭터와 그래픽

### 그래픽 스타일

- **Nidhogg 1 (2014) 수준의 미니멀 픽셀 아트**
- 캐릭터는 단색 실루엣에 가까운 수준
- 맵도 심플한 픽셀 배경 (단색 그라데이션 + 간단한 지형)
- 디자인에 공을 많이 들이지 않아도 되는 방향

### 캐릭터 모션

- **몸통:** 거의 움직이지 않음. 서 있거나, 달리거나, 점프하는 정도
- **팔:** 마우스 조준 방향으로 팔이 움직임 (ROUNDS 스타일)
- **좌우 전환:** 왼쪽 ↔ 오른쪽 방향 전환 시 부드러운 전환 모션 적용
- 복잡한 액션 모션(구르기, 날라차기 등)은 **없음**

## 6. 개발 환경

### 어떤 엔진을 써야 할까?

이 게임은 **2D 픽셀 그래픽 + 실시간 물리(총알 궤적, 충돌) + 멀티플레이어**가 핵심이에요.
웹(Canvas/WebGL)으로도 못 만들 건 아니지만, 게임 엔진을 쓰는 게 압도적으로 효율적이에요.

| 항목 | Unity | Godot |
|---|---|---|
| **언어** | C# | GDScript (Python 비슷) / C# |
| **2D 지원** | 좋음 (근데 3D 엔진 기반) | 2D 전용 엔진이 따로 있을 정도로 강력 |
| **픽셀아트 친화** | 설정 좀 만져야 함 | 기본 설정이 픽셀 친화적 |
| **멀티플레이어** | Netcode for GameObjects, Mirror 등 | 자체 멀티플레이어 API 있음 |
| **학습 곡선** | 중간 (C# 경험 있으면 낮음) | 낮음 |
| **배포** | Steam, 모바일, 웹 등 다양 | Steam, 모바일, 웹 등 다양 |
| **커뮤니티/자료** | 압도적으로 많음 | 빠르게 성장 중이지만 Unity보다는 적음 |

### 추천

**"처음 게임 개발을 해본다"면 → Godot 4**

- GDScript가 Python이랑 비슷해서 진입 장벽이 낮아요
- 2D 게임 만들기에 특화되어 있어서, 이 게임 컨셉에 딱 맞아요
- 오픈소스라 무료이고, 가볍고, 빌드도 빠름

**"향후 포트폴리오나 확장성을 생각한다"면 → Unity**

- 게임 업계 표준에 가깝고, 자료가 많아서 막혔을 때 답을 찾기 쉬움
- C#이 Java랑 문법이 비슷해서 적응이 빠를 거예요
- Steam 배포 파이프라인이 잘 정리되어 있음

### 멀티플레이어 구현

온라인 멀티플레이어 기준으로 설계해요. 마우스 조준이 핵심인 게임이라 같은 PC에서 두 명이 플레이하는 건 구조적으로 어려워요.

- Godot의 경우 `MultiplayerAPI`, Unity의 경우 `Mirror` 또는 `Netcode for GameObjects` 활용

### 기술 스택 요약

```
게임 엔진:    Godot 4 (추천) 또는 Unity
언어:         GDScript 또는 C#
그래픽:       Aseprite (픽셀 아트 제작 툴)
사운드:       Audacity / BFXR (효과음 생성)
버전 관리:    Git + GitHub
배포 타겟:    Steam (PC)
```

## 7. 개발 우선순위 (제안)

큰 그림을 단계별로 나누면 이렇게 돼요.

**1단계: 프로토타입 (코어 루프)**
- 가로로 긴 맵 하나, 두 캐릭터 배치
- 이동, 점프, 마우스 조준, 발사
- 총알 충돌 → 사망 → 리스폰
- 상대 진영 도달 시 승리 판정

**2단계: 카드 시스템**
- 사망 시 카드 선택 UI
- 카드 효과 구현 (유도, 산탄, 고속, 대구경 등)
- 카드 중복 적용 로직

**3단계: 카메라 & 게임 흐름**
- 이기는 쪽 기준 카메라 추적
- 카메라 이탈 시 자동 사망 처리
- 리스폰 위치 로직 정리

**4단계: 비주얼 & 사운드**
- 픽셀 아트 캐릭터/맵 제작
- 방향 전환 모션
- 효과음, BGM

**5단계: 멀티플레이어**
- 온라인 멀티플레이어 구현
- 네트워크 동기화 (총알 위치, 플레이어 상태 등)

## 8. 이 게임이 재밌는 이유

1. **역전의 쾌감:** 밀리고 있을수록 총이 강해져서, 한 방에 전세가 뒤집힐 수 있어요
2. **긴장감:** 제한 시간이 없어서 느긋할 것 같지만, 상대가 달려가는 걸 보면 심장이 쫄깃해요
3. **카드 조합의 재미:** 유도탄 + 산탄 + 대구경 = 커다란 유도 산탄이라는 괴물이 탄생
4. **단순한 조작:** 이동 + 점프 + 조준 + 발사. 이게 끝. 근데 깊이가 있음
5. **짧은 판:** 한 판이 1~5분 내외. 부담 없이 "한 판 더"가 나옴
